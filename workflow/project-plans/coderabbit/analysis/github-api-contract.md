# GitHub API Contract for PR Follow-Through

Plan: `PLAN-20260429-CODERABBIT-PR-FOLLOWUP`
Phase: `P02`

This contract is the only external GitHub data source inventory that PR follow-through implementation phases may consume. Every row names the exact command or query, fixture file, JSON pointer, and assertion name required by the Phase 02 validator.

## Contract validation command

Run:

```bash
cargo run --bin github_api_contract_validator -- project-plans/coderabbit/analysis/github-api-contract.md tests/fixtures/github_api_contract
```

## CodeRabbit readiness truth table and fixture mapping

| Row | Inputs and precedence | Fixture mappings | Expected artifact state | Assertions |
|---|---|---|---|---|
| Ready empty after stable current-head signal | Current-head completed CodeRabbit check or review signal exists, no in-progress signal exists, item set is empty for two identical observations | `coderabbit_readiness_ready_empty.json`: `/source`, `/head_sha`, `/bot_login`, `/status`, `/conclusion`, `/summary_body`, `/items` | `coderabbit-feedback.json.readiness_state=ready`, `items=[]`, `stable_observation_count=2`, `outcome_reason=stable_ready_empty` | `github_api_contract::readiness_ready_empty` |
| In-progress overrides ready text | In-progress CodeRabbit check state on current head overrides review/comment text that appears complete | `coderabbit_readiness_in_progress.json`: `/source`, `/head_sha`, `/bot_login`, `/status`, `/conclusion`, `/summary_body` | `readiness_state=not_ready`, `signals_seen.in_progress=true`, no clean completion | `github_api_contract::readiness_in_progress_overrides` |
| Stable ready with feedback | Two consecutive ready observations have identical normalized item-set hash and current-head feedback items | `coderabbit_readiness_stable_ready.json`: `/observations/0/ready_signal`, `/observations/0/feedback_item_set_hash`, `/observations/1/ready_signal`, `/observations/1/feedback_item_set_hash`, `/observations/1/items_count` | `readiness_state=ready`, `stable_observation_count=2`, `items_count=2` | `github_api_contract::readiness_stable_ready` |
| Timeout after absent signal | Six observations lack ready signal and no supported disabled configuration is set | `coderabbit_readiness_timeout.json`: `/observations/0/ready_signal`, `/observations/5/ready_signal`, `/observations/5/items_count` | `readiness_state=timeout`, `budget_used=6`, `budget_remaining=0`, `failure_sequence` allocated | `github_api_contract::readiness_timeout_budget` |
| API fatal or partial required-surface failure | Required GraphQL or REST surface returns forbidden, malformed, or partial data that prevents readiness confidence | `permission_denied_graphql.json`: `/errors/0/type`, `/metadata/operation`, `/metadata/required_scope_or_permission`, `/metadata/account_login`, `/metadata/token_type` | `readiness_state=fatal`, `failure_reason=permission_denied`, permission metadata copied | `github_api_contract::readiness_permission_denied` |
| Non-CodeRabbit bot noise | Feedback surfaces include authors outside configured identity set | `issue_comments_rest_page2.json`: `/0/user/login`, `/0/body` | Noise is recorded in observation metadata and excluded from `items` unless identity matches configuration | `github_api_contract::readiness_filters_non_coderabbit_noise` |

The Phase 01 artifact field names remain valid after mapping to API fixtures. External fields are normalized into the existing artifact names as follows:

| Artifact field | External source paths |

| `pr.json.head_sha` | `pr_identity_gh_pr_view.json`: `/headRefOid`; `pr_identity_graphql_fallback.json`: `/data/repository/pullRequest/headRefOid` |

| `pr-feedback-marker-report.json.resolved_threads[].thread_id` | Resolve mutation `/data/resolveReviewThread/thread/id` |
