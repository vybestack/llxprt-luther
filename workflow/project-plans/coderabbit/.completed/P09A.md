## Semantic Verification

- Exactly one accepted validated result per current feedback item on success: `evaluate_coderabbit_feedback` computes `complete` using `exactly_one_accepted_per_item(&items, &accepted_results)` before returning `StepOutcome::Success`; otherwise it returns fatal. See `src/engine/executors/feedback_eval.rs` lines 485-546 and 871-900.
- One `FeedbackEvaluationRequest` per item per attempt, no batch prompts/responses accepted: the executor builds one request inside the per-item retry loop (`build_request`) and calls `adapter.evaluate(&request)` once per attempt. `validate_response` rejects arrays, batch fields, extra identity fields, and any array-valued response field. See lines 411-469 and 623-654. Test `feedback_evaluation_single_item_request_rejects_batch_wrong_hash_unknown_decision_and_missing_reason` asserts three single-item requests and batch rejection.
- Response validation rejects malformed JSON, arrays, extra/multiple item IDs, wrong item ID/key/body hash/head SHA, unknown decisions, and missing required reasons/actions: `validate_response` parses strict JSON and returns rejection reasons for malformed JSON, arrays/batch fields, wrong identity/binding fields, unknown decisions, missing non-valid reasons, and missing recommended action. See lines 614-704. Tests at `tests/github_pr_followup_executor_tests.rs` lines 1812-1908 and 2146-2227 cover array/batch, wrong hash/head, and extra identities; shell-safety test covers malformed free-form JSON.
- Retry cap is strictly internal/fixed at 3 attempts per item: `MAX_ATTEMPTS_PER_ITEM` is a constant at line 24 and `evaluate_coderabbit_feedback` assigns `let max_attempts = MAX_ATTEMPTS_PER_ITEM;` at line 328. Negative grep confirms no `max_attempts_per_item` step param ingestion. Test `feedback_evaluation_ignores_max_attempts_param_override` sets the param to 4 and proves only 3 calls are made.
- Budget exhaustion is not encoded as an accepted decision: if no accepted response is obtained, the item is appended to `budget_exhausted_items` only, and `complete` requires empty budget exhaustion plus exactly one accepted per item. Tests assert accepted results length is zero on budget exhaustion.
- Output separates `accepted_results`, `rejected_attempts`, `unevaluated_items`, and `budget_exhausted_items`: `FeedbackEvaluationArtifact` has separate fields at lines 301-312 and the executor populates them independently.
- Prior accepted unchanged evaluations are strictly binding-validated, reused without adapter invocation, emitted once, and malformed/ambiguous/stale/missing-binding prior state is fatal rather than silent overwrite/fresh fallback: `reusable_evaluation` enforces duplicate/stale/malformed fatal paths and calls `validate_reusable_accepted`, which requires item identity plus repository owner/name and PR number. Tests `feedback_evaluation_reuses_unchanged_accepted_state_without_reinvoking_adapter` and `feedback_evaluation_rejects_unbound_reusable_state_as_fatal` pass.
- New accepted evaluations persist back to state/history before success: accepted results are added via `upsert_state_entry`; when `complete`, `write_state_artifact` writes `coderabbit-feedback-state` before `write_evaluation_artifact` and before returning `StepOutcome::Success`. See lines 441-456 and 511-546.
- Shell-safety test exercises evaluator command/process seam: `CommandFeedbackEvaluationAdapter` serializes the structured request to JSON stdin and invokes `FeedbackEvaluatorCommandRunner` with argv. The production runner uses `Command::new(program).args(args)` and stdin, not a shell. Test `feedback_evaluator_command_shell_safety_writes_raw_llm_text_to_bounded_artifacts` asserts argv is fixed, request/body text is stdin JSON, malicious LLM text is not in argv, and malformed raw text is rejected.
- Raw LLM output/audit artifacts are written through `PrFollowupArtifactStore`, not direct file writes in `feedback_eval.rs`: `write_raw_response` calls `store.write_raw_text_artifact(...)`; negative grep found no direct `std::fs` write/create/remove/rename/copy mutation in `feedback_eval.rs`.
