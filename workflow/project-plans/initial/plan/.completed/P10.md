## Implementation Highlights

### 10A: Runtime Paths
- `get_data_dir()` - Returns platform-specific data directory
- `get_config_dir()` - Returns platform-specific config directory
- `get_run_dir(run_id)` - Returns run-specific directory path
- `get_artifacts_root()` - Returns artifacts root path

### 10A: Git Adapter
- `RepositoryConfig` - Configuration for source, workspace, branch template
- `resolve_workspace_path()` - Handles shared/per_run workspace strategies
- `prepare_branch()` - Creates/checkout branches with template substitution
- `push_branch()` - Pushes branches to remotes
- `clone_repository()` - Clones source to destination
- `init_repository()` - Initializes new git repository

### 10B: Monitor Process
- `SingletonGuard`/`SingletonLock` - File-based singleton lock with auto-cleanup
- `acquire_singleton_lock()` - Acquires lock with stale lock detection
- `calculate_backoff()` - Exponential/fixed backoff calculation
- `ProcessState` - Tracks restart attempts with backoff
- `is_process_alive()` - Platform-specific PID checking

### 10B: Heartbeat
- `Heartbeat` struct - Instance ID, timestamp, state, metadata
- `write_heartbeat()` - Async heartbeat persistence to JSON
- `read_heartbeat()` - Reads heartbeat from disk
- `HeartbeatWriter` - Periodic heartbeat writer with state tracking
- `MonitorState` enum - Starting, Running, Degraded, Stopping, Stopped, Error

### 10B: IPC
- `create_ipc_endpoint()` - Creates Unix socket at unique path
- `serve_status()` - Serves IPC requests (status, ping, heartbeat, shutdown)
- `IpcRequest`/`IpcResponse` - Typed IPC protocol
- `connect_ipc()` - Connect to IPC endpoint
- `send_request()` - Send request and receive response

### 10C: Service
- `ServiceSpec` - Binary path, args, working dir, environment variables
- `generate_launchd_plist()` - Generates macOS launchd plist XML
- `generate_systemd_unit()` - Generates systemd unit file
- `install_launchd_service()` - Installs to ~/Library/LaunchAgents/
- `install_systemd_service()` - Installs to ~/.config/systemd/user/
- Service control: start/stop/uninstall/status check

### 10D: Wiring
- All modules exported from `lib.rs`
- CLI skeleton in `main.rs` for monitor/service modes
