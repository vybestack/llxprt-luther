### Nine-stage viability gate (each canary traverses all nine in exact order)

1. **Atomic fresh launch / capsule** — `persist_launch_atomically` (Starting
   RunMetadata + immutable ExecutionCapsuleV1 in one IMMEDIATE tx); capsule
   envelope digest verifies; immutability checked.
2. **Deliberate interruption after worktree delta** — worktree delta file
   written; run left in interrupted (Running) state.
3. **Supported protocol recovery** — `RecoveryProtocolV1::recover_with_executor`
   dispatches prepare → reserve → execute → finalize; injected truthful
   executor returns deterministic result; outcome is `Recovered`.
4. **ContinueWorkspace ownership/base/workspace verification** — durable
   ownership marker persists; capsule `base_ref` verified; workspace identity
   intact through reserve revalidation.
5. **Allowlisted staging adapter** — staging via `prepare_effect` (Commit
   kind); the delta file is the allowlisted path; intent is `prepared`.
6. **Commit/push effect intents with reconcile/no duplicates** — commit
   reconciles to `Completed` (HEAD matches target); idempotent re-prepare
   returns completed (no duplicate row); push prepares + reconciles to
   `Completed`; exactly two intents (commit + push), no duplicates.
7. **PR identity binding** — repository + pr_number + head_sha bound on
   RunMetadata.
8. **Stable final-head CI/review** — run transitions to ReviewReady; injected
   remote probe observes merge as completed with correct strategy;
   `build_reachability_proof` produces strategy-specific proof.
9. **Strategy-specific typed merge artifact + Merged** —
   `complete_merge_required_run` derives exact base from capsule.base_ref,
   builds artifact from independently computed verifier evidence, commits
   artifact + ReviewReady → Merged atomically; `completion_satisfied` requires
   both artifact AND Merged; artifact proof matches strategy.
