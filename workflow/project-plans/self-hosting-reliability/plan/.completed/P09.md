# Phase P09 Completion

Phase: `PLAN-20260723-SELFHOST-RELIABILITY.P09`
Completed: 2026-07-23
Verdict: PASS

Created `src/engine/recovery/protocol.rs` exposing the single typed recovery
abstraction `RecoveryProtocolV1` with its phased model (prepare → reserve →
execute → finalize). Public `RecoveryRequest` carries `run_id`, `step_id`,
`expected_epoch`, and `operator_verb` — NO authorization bool (authority is
derived internally via the sealed `RecoveryAuthority`). [C4/B2]

`RecoveryAuthority` and `PreparedRecovery` are sealed: their fields are private
and only constructible inside the protocol module (private `new` constructors
plus a private `Sealed` token), so code outside the module cannot forge them.
[C4] `PreparedRecovery` retains a `VerifiedWorkspace` anchor and captures the
exact durable authority snapshot (`run_status`, `current_step`, `live_pid`,
`checkpoint_identity`, `wait_state`, `lease`) by consuming the durable stores
directly (`RunStatus`, `WaitStateRecord`, `IssueLease`, `AttemptRow`,
`ExecutionCapsuleV1`). [B1/B6]

`RecoveryOutcome` includes `Recovered`, `AlreadyApplied { prior_outcome,
attempt_id, operation_id }`, `Refused`, `StaleEpoch`, and `Conflict`.
`RefusalReason` is aligned with P09A semantics including
`ConflictingOperation`. `OperatorVerb`, `RecoveryStrategy`, and `RecoveryError`
are defined per the locked P02 pseudocode.

`recover` is stubbed with `todo!()` (designated P11 behavior). `policy_for_step`
returns `NonRecoverable` (fail-closed default) and `select_strategy` returns
`Refused(NonRecoverable)` (fail-closed default) until P11. No `// TODO`/`//
FIXME` comments; no suppressions; no duplicate type definitions
(`RecoveryStrategy`/`RefusalReason` moved from `policy.rs` to `protocol.rs` and
re-exported). No in-memory persistence facade.
