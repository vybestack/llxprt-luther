## Semantic Verification (code-read audit)

### execute()
- Reads `checks` array from params and errors if missing/non-array.
- Returns immediate Success for empty checks and sets:
  - `verify_passed=true`
  - `verify_summary="No checks configured"`
- For each check:
  - Resolves command via `resolve_check_command()`.
  - Runs real shell command via `Command::new("sh").arg("-c")...spawn()` in `context.work_dir()`.
  - Captures stdout/stderr and exit code.
  - Parses output via `parse_check_output()`.
  - Appends `CheckResult { check_type, passed, exit_code, errors, raw_stdout, raw_stderr }`.
- Creates report summary via `build_summary()`.
- Writes JSON report to `.luther/verify-report.json` via `serde_json::to_string_pretty` + `std::fs::write`.
- Sets context vars:
  - `verify_passed`
  - `verify_summary`
  - per-check error vars: `test_failures`, `build_errors`, `type_errors`, `lint_errors`, `format_errors`.
- Returns:
  - `Success` when all checks pass
  - `Fixable` when any check fails
  - `Fatal` on spawn/wait failures and command-not-found conditions.

Result: [OK] Meets required behavior.

- Checks `params.check_commands[check_type]` first; returns custom command when present.
- Otherwise returns defaults for `lint`, `typecheck`, `test`, `format`, `build`.
- Unknown type returns `EngineError::StepExecutionError`.

Result: [OK] Custom override precedence is implemented.
