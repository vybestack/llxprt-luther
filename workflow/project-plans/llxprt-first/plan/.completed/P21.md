## Design Decisions

1. **SmokeTestExecutor Pattern**: The hybrid executor approach allows the first 3 steps to run real commands (gh, git) while returning Fatal for remaining steps. This triggers the fatal transition to `abandon_and_log`, which also runs real cleanup commands (unassign, unlabel).

2. **Config Resolution**: All config values loaded from TOML via the resolution system - no hardcoded repo names, assignees, or paths in the test source.

3. **TempDir Isolation**: `work_dir` overridden with a fresh `TempDir` for each test run, ensuring test isolation.

4. **Ignored Tests**: Both tests marked with `#[ignore]` because they require:
   - `gh` CLI authentication
   - Network access to GitHub
   - Permission to modify real GitHub issues (assign, label, comment)
