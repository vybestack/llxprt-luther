### Implementation Details

#### 1. Modified `StepContext::new()`
- Added automatic storage of built-in variables (`work_dir`, `run_id`) in flat `variables` HashMap
- Ensures bare key access to built-ins works without namespace (REQ-LF-CTX-004)

#### 2. Modified `StepContext::set()`
- Implements dual-write pattern:
  - If `current_step_id` is `Some(step_id)`: stores in `namespaced_vars[step_id][key]`
  - ALWAYS stores bare key in `self.variables` for backward compatibility
- Respects the requirement to store in both namespaces simultaneously

#### 3. Modified `StepContext::get()`
Implements 4-case resolution order:
1. **Case 1**: Explicit namespace "step_id.variable" - splits on first '.', looks up in `namespaced_vars[step_id][var]`
2. **Case 2**: Unnamespaced - iterates `step_order` in REVERSE (most-recent-writer-first), checking each step's `namespaced_vars` for the key
3. **Case 3**: Falls back to "config" namespace for config-seeded variables (loaded at run start by Phase 15)
4. **Case 4**: Falls back to flat `variables` for built-ins and pre-namespace-era bare keys

This correctly implements REQ-LF-CTX-002's "most-recent-writer-first search across all step namespaces."

#### 4. Modified `interpolate_string()`
- Collects all resolvable keys including:
  - Namespaced keys: "step_id.var" from `namespaced_vars`
  - Bare variable names from namespaced vars
  - Keys from flat `variables`
  - Built-in keys: `work_dir`, `run_id`
- Sorts keys by length descending to prevent partial replacement
- Delegates resolution to `get()` for both `{step_id.var}` and `{var}` placeholders
- Leaves unresolved placeholders as-is (backward compatible)
