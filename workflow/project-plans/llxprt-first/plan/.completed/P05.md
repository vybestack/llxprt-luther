### Functions Implemented

#### 1. `extract_dot_path()`
- Splits dot-path on `.`
- Walks JSON tree segment by segment
- Returns `None` if any segment is missing
- Handles leading dots correctly

#### 2. `json_value_to_string()`
- `String` → returns inner string
- `Number`/`Bool` → `to_string()`
- `Array`/`Object` → `serde_json::to_string()`
- `Null` → empty string `""`

#### 3. `parse_outcome_name()`
- Maps "success" → Success
- Maps "fixable" → Fixable
- Maps "fatal" → Fatal
- Maps "retryable" → Retryable
- Maps "abandon" → Abandon
- Unknown → defaults to Success

#### 4. `execute()` method (rewritten)
- **Stdin handling**: `stdin` (interpolated string) and `stdin_file` (file contents) params
- **Command spawn**: Uses `child` pattern for stdin piping
- **Exit code evaluation**: Follows strict order per spec:
  1. Non-zero exit → check `exit_code_map` first → return mapped outcome OR default Fixable
  2. Zero exit + `outcome_on_stdout` → scan for patterns → return first match
  3. Zero exit + `output_format == "json"` → parse JSON, extract via `context_map`
  4. Default → Success
- **JSON parsing**: Full error handling with `json_parse_error` context variable
- **Missing path error**: Sets `json_path_error` with available top-level keys
