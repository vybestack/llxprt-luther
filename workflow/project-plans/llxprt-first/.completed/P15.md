## Critical Changes

### Fatal Routing (REQ-LF-FAIL-001)

**Before**: `StepOutcome::Fatal` immediately returned `RunOutcome::Failure`, bypassing the transition table.

**After**: All outcomes (including Fatal) flow through `resolve_next_step()`. If no transition is found:
- Fatal → RunOutcome::Failure (backward-compatible fallback)
- Success → RunOutcome::Success (terminal step)
- Fixable → RunOutcome::Failure (no recovery path)

This enables TOML workflows to define `condition = "fatal"` transitions that route to cleanup steps.
